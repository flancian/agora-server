*Compiled by Antigravity (AGY) on 2026-05-22*

This document synthesizes the goals, architecture, tasklists, and future direction of the Agora project across `~/agora`, `~/agora-server`, and `~/agora-bridge`. It serves as a unified map for our ongoing collaboration.

---

## 1. Project Philosophy & Core Goals

The Agora is a **Free Knowledge Commons** and an experimental distributed knowledge graph built "for the benefit of all beings."

*   **Radical Sovereignty & Decentralization:** The filesystem is the ultimate source of truth. The Agora acts as a lens, not a silo, respecting users' ownership of their digital gardens.
*   **The Agora Protocol:** 
    *   **Nodes are Concepts:** Abstract shared topics (e.g., `[[Love]]`).
    *   **Subnodes are Utterances:** Concrete individual contributions (e.g., `@flancian/love.md`).
    *   **No 404s:** Every missing link is an invitation to contribute.
*   **Polite Software:** Development must be holistic, focusing on both code and poetry, ensuring the system is unobtrusive, resilient, and empowering.

## 2. Architecture: The Server/Bridge Split

To balance real-time responsiveness with heavy background processing, the Agora employs a hybrid architecture communicating via a shared SQLite database (`agora.db`).

*   **Agora Server (`agora-server`):** The front-end Flask application.
    *   **Role:** Read-mostly, request-handling layer.
    *   **Indexing:** "Hot" (on-demand) indexing of recently accessed files.
    *   **Priority:** Stability, speed, and protecting the monolith's memory footprint via shared object identity and zero-downtime rolling restarts (uWSGI chain reloading).
*   **Agora Bridge (`agora-bridge`):** The background microservice/worker.
    *   **Role:** Write-heavy, asynchronous processing layer.
    *   **Indexing:** "Cold" (batch) indexing, full-content indexing.
    *   **Responsibilities:** Repository pulling (gardens, stoa), ActivityPub federation pushing, and running social media bots (Mastodon, Twitter, Bluesky).

## 3. Consolidated Roadmap & Open TODOs

Based on `ROADMAP.md`, `NEXT.md`, and code context, these are the outstanding initiatives:

### Data, Search & Indexing
*   **SQLite as Primary Source (Server Priority 2):** Transition server data-fetching logic to read-only SQLite queries, eliminating filesystem walks during HTTP requests.
*   **FTS5 Migration (Bridge Priority 1 & Server Priority 6):** Implement SQLite FTS5 for instant full-text search, replacing slow Python iteration and adding trigram indexing for fuzzy search (handling typos gracefully).
*   **Semantic Search / The Compass (Server Priority 5):** Integrate `sqlite-vec` to store embeddings generated by the Bridge, enabling nearest-neighbor semantic search to connect conceptually related thoughts.

### Federation & Social Web (Bridge Priority 2)
*   **ActivityPub Real-Time Push:** Ensure the Bridge processes the federation queue to push signed `Create` activities to followers' inboxes. (Verification is pending).
*   **AT Protocol / Bluesky Integration:** Build out native support for Bluesky, potentially via an Agora-specific XRPCSkeleton (Feed Generator) to let people subscribe directly.

### The Read-Write Web (Collaboration & UX)
*   **"Fork to Garden" (The Siphon) (Server Priority 4):** Allow users to pull knowledge out of the Commons into their own sovereign garden (e.g., redirecting to the Bullpen editor), inspired by Ward Cunningham's Federated Wiki.
*   **Conditional Autopull (Server Priority 3):** Automatically enrich empty nodes by pulling Wikipedia or Fediverse content if a user opts in via settings.
*   **Go Links UX (Server Priority 2):** Improve the UI for `[[go]]` links with visual highlights and stateful tooltips.

### Infrastructure & Operations
*   **Bullpen Editor Deployment (Next 1):** Fully deploy `bullpen.py` and `edit.anagora.org` routing so users can edit their files securely. (We recently laid the groundwork by making the Editor URL customizable).
*   **Bot Status Observability:** Add health checks and reporting for the Mastodon, Bluesky, and Twitter bots to the Bridge's API dashboard (`/`).
